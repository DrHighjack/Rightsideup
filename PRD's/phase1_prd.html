02

User Roles

Two roles for Phase 1. TC role schema can be stubbed but not implemented.

Realtor
  • Self-registers with email + password
  • Can place, view, and cancel their own orders
  • Cannot see other realtors' data
  • Cannot access admin area
  • Sees only their own order history
Admin
  • Created manually in database (no self-signup)
  • Full read/write on all orders and clients
  • Can create orders on behalf of any realtor
  • Can edit order status, details, notes
  • Access to admin-only routes and UI

Admin accounts must be seeded directly in the database. There should be NO public-facing route to register as an admin. The role field on the User table should never be settable via the API from the client side.

03

Authentication

Use NextAuth.js with Credentials provider (email + password). Passwords hashed with bcrypt.

Registration flow (realtors only)

1
User visits /register — form collects: first name, last name, email, phone, brokerage name, password, confirm password
2
Validate: email unique, password ≥ 8 chars, all fields required
3
Hash password with bcrypt (salt rounds: 12), save User record with role = "REALTOR"
4
Send welcome email (basic transactional email via Resend or Nodemailer)
5
Redirect to /dashboard with active session

Login flow

1
User visits /login — form: email, password
2
NextAuth Credentials provider validates against DB, bcrypt.compare()
3
On success: session created, redirect based on role — REALTOR → /dashboard, ADMIN → /admin
4
On failure: show inline error "Invalid email or password" — do not specify which is wrong

Session & protection

  • All /dashboard/* routes require REALTOR or ADMIN session — redirect to /login if unauthenticated
  • All /admin/* routes require ADMIN session — redirect to /dashboard if authenticated as REALTOR
  • All /api/* routes validate session server-side before executing
  • Session JWT stored in httpOnly cookie, 30-day expiry
  • Include role in NextAuth session callback so middleware can check it

05

API Routes

All under /api/. All require valid session. All return JSON.

Auth

POST/api/auth/registerCreate realtor account. Body: email, password, firstName, lastName, phone, brokerageName
POST/api/auth/[...nextauth]NextAuth handler — login, logout, session

Orders — Realtor

GET/api/ordersReturn all orders for the logged-in realtor. Query params: status, type, page, limit
GET/api/orders/[id]Single order. Realtors can only fetch their own. Admins can fetch any.
POST/api/ordersCreate new order. Body: type, address, addressLat, addressLng, scheduledDate, notes
PUT/api/orders/[id]/cancelCancel an order. Realtors can only cancel PENDING orders. Sets status = CANCELLED, records cancelReason.

Orders — Admin only

GET/api/admin/ordersAll orders across all realtors. Supports filters: status, type, realtorId, date range, search. Paginated.
POST/api/admin/ordersAdmin creates order on behalf of a realtor. Same body as realtor POST + realtorId field.
PUT/api/admin/orders/[id]Update any order field including status, scheduledDate, adminNotes, type. Full edit access.
DEL/api/admin/orders/[id]Hard delete. Admin only. Require confirmation flag in body: { confirm: true }
GET/api/admin/usersList all realtor accounts. Supports search by name/email/brokerage. No passwordHash returned.
GET/api/admin/users/[id]Single user detail with their order history summary.

PWA Configuration

Makes the site installable on iPhone and Android as a home screen app.

manifest.json
  • name: "SignPost Field"
  • short_name: "SignPost"
  • display: "standalone"
  • start_url: "/dashboard"
  • theme_color: brand green
  • background_color: white
  • Icons: 192x192 and 512x512 PNG
Service worker
  • Use next-pwa package
  • Cache static assets and shell
  • Offline fallback page: "You're offline — reconnect to place orders"
  • Do not cache API calls (dynamic data)
iOS meta tags
  • apple-mobile-web-app-capable: yes
  • apple-mobile-web-app-status-bar-style: black-translucent
  • apple-mobile-web-app-title: SignPost

Acceptance Criteria

Phase 1 is done when all of these pass.

AuthA new realtor can register, log in, and be redirected to their dashboard. An admin logging in goes to /admin. A realtor cannot access /admin.
Order — placeRealtor can place an Install, Removal, or Change order with a Maps-autocompleted address, date, and optional notes. Order appears in their list with status PENDING immediately.
Order — emailA confirmation email is sent to the realtor with their order number and details within 30 seconds of placing an order.
Order — cancelRealtor can cancel a PENDING order from its detail page. Cancelled orders show in list with CANCELLED status. Orders in other statuses show no cancel button.
Admin — viewAdmin can see all orders from all realtors in /admin/orders with filters working correctly for status, type, and realtor name search.
Admin — editAdmin can change the status of any order, edit all fields, add admin notes, and those changes are reflected immediately in the realtor's view (minus adminNotes).
Admin — createAdmin can create an order on behalf of any existing realtor account.
PWAOn iPhone Safari, visiting the URL and tapping "Add to Home Screen" installs the app with the correct icon and name. It launches full-screen with no browser chrome.
Mobile UXAll realtor-facing pages are fully usable on a 390px wide screen (iPhone 14). Bottom tab nav is present and functional. No horizontal scrolling on any page.
SecurityUnauthenticated requests to /api/orders return 401. Realtor requests to /api/admin/* return 403. PasswordHash is never present in any API response.
