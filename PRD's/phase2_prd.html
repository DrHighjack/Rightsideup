02

QuickBooks Integration

Use the QuickBooks Online API (v3). OAuth 2.0 flow initiated by admin. One QBO company connected to the whole app — not per-realtor.

Use the node-quickbooks npm package (intuit-oauth for the OAuth flow, node-quickbooks for API calls). Register your app at developer.intuit.com — you'll get a sandbox company to test with before going live. Sandbox credentials are separate from production.

OAuth connection flow

1
Admin visits /admin/settings/quickbooks — page shows "Connect to QuickBooks" button if not yet connected
2
Button initiates OAuth: redirects to Intuit's auth page. Scopes needed: com.intuit.quickbooks.accounting
3
Intuit redirects back to /api/quickbooks/callback with auth code + realmId (company ID)
4
Exchange code for access_token + refresh_token. Store both in a QBOConnection record in DB (encrypted at rest). Store realmId.
5
Admin settings page now shows "Connected — [Company Name]" with a Disconnect button
6
Access tokens expire in 1 hour. Refresh tokens expire in 100 days. Build a token refresh helper that automatically refreshes before each API call if token is within 5 min of expiry.

Auto-invoice on order completion

1
Admin changes order status to COMPLETED via the admin order edit page
2
Server checks: does an Invoice already exist for this order? If yes, skip. If no, proceed.
3
Look up or create a QBO Customer record for the realtor (match by email). Store QBO CustomerId on the User record.
4
Create QBO Invoice with: line item = order type + address, amount from service pricing, due date = 30 days out, customer = realtor
5
Save Invoice record in local DB with qboInvoiceId, status = SENT, amount, dueDate, linked orderId and realtorId
6
QBO handles emailing the invoice to the realtor natively — no extra email code needed

Sync invoice status from QBO

  • Add a /api/quickbooks/webhook endpoint to receive QBO webhook events
  • Register webhook in QBO developer portal for Invoice events (create, update, delete)
  • On webhook: verify HMAC signature, parse event, update local Invoice status if payment received
  • Also add a manual "Sync Now" button in admin invoices that polls QBO for status updates on all open invoices
  • Invoice statuses to map: QBO "Paid" → local PAID, QBO "Voided" → local VOIDED

New API Routes

QuickBooks

GET/api/quickbooks/connectInitiates OAuth flow — redirects to Intuit auth URL. Admin only.
GET/api/quickbooks/callbackOAuth callback. Exchanges code for tokens, saves QBOConnection, redirects to /admin/settings/quickbooks.
POST/api/quickbooks/disconnectRevokes token, deletes QBOConnection. Admin only.
POST/api/quickbooks/webhookReceives QBO webhook events. Verifies HMAC. Updates local invoice status.
POST/api/quickbooks/syncManual sync — polls QBO for status of all open invoices. Admin only.

Invoices

GET/api/invoicesRealtor: their invoices. Query params: status, page. Returns amount, status, dueDate, orderId.
GET/api/invoices/[id]Single invoice detail. Realtor can only fetch their own.
GET/api/admin/invoicesAll invoices. Filters: status, realtorId, date range. Includes totals summary.
POST/api/admin/invoicesManually create invoice. Body: realtorId, orderId?, amountCents, dueDate, notes, couponCode?
PUT/api/admin/invoices/[id]Update invoice — status, notes, dueDate. Voiding also voids in QBO.

Coupons

POST/api/coupons/validateRealtor submits a code. Returns: valid/invalid, discount type, value, expiry. Does not consume the coupon.
GET/api/admin/couponsAll coupons. Admin only.
POST/api/admin/couponsCreate coupon. Body: code, type, value, maxUses, restrictToUserId, expiresAt.
PUT/api/admin/coupons/[id]Edit or deactivate coupon. Can toggle isActive.
DEL/api/admin/coupons/[id]Delete coupon — only if usedCount is 0. Otherwise deactivate instead.

Client tracker (admin)

PUT/api/admin/users/[id]Update adminNotes, tags on a client record. Cannot change role or password via this route.
GET/api/admin/users/[id]/summaryReturns: order count, total invoiced, outstanding balance, last order date, all invoices.

SMS — internal only

POST/api/internal/smsInternal server-side helper — not called by client. Sends SMS via Twilio. Called by order status change logic. Checks smsOptIn before sending.
05

Twilio SMS Notifications

Triggered server-side on order status changes. Use twilio npm package.

Order → SCHEDULED"Your sign install at [address] is scheduled for [date]. Order #[number] — SignPost Field"
Order → COMPLETED"Your sign at [address] has been installed. Invoice on the way. Order #[number] — SignPost Field"
Order → CANCELLED (by admin)"Your order #[number] at [address] was cancelled. Log in to place a new order or contact us."
Invoice → SENT"Invoice #[number] for $[amount] is ready. Due [date]. View in your SignPost portal."

Only send SMS if the realtor's smsOptIn is true AND they have a phone number on file. Always check both conditions before calling Twilio. Add an opt-out link or instruction in each message per carrier compliance rules.

Acceptance Criteria

QBO connectAdmin can connect to a QuickBooks sandbox company via OAuth. Settings page shows company name and "Connected" after auth.
Auto-invoiceWhen admin marks an order COMPLETED, an invoice is created in QBO within 5 seconds and a matching Invoice record appears in the local DB and in the realtor's invoice list.
No duplicate invoicesMarking an already-completed order COMPLETED again does not create a second invoice.
Invoice syncWhen an invoice is paid in QBO, the local status updates to PAID either via webhook or manual sync.
Coupon — validRealtor enters a valid, active coupon code on an invoice. Discount is calculated and displayed before the invoice is saved. Coupon usedCount increments by 1.
Coupon — invalidExpired, inactive, maxed-out, or non-existent codes return a clear error message. No discount is applied.
SMS — sentWhen order status changes to SCHEDULED or COMPLETED, a Twilio SMS is sent to the realtor's phone within 10 seconds, IF smsOptIn is true and phone is set.
SMS — opt outRealtor with smsOptIn = false receives no SMS on any status change.
Client trackerAdmin can view a realtor's full order and invoice history from /admin/clients/[id], see their outstanding balance, add notes, and add/remove tags.
Token securityQBO access_token and refresh_token are encrypted in the database. They are never returned in any API response to the client.
