Step 1 — Fix OAuth flow

The most likely reason QB auth failed before: the redirect URI in developer.intuit.com didn't match. Now that you have https://app.northshoresignco.com, add https://app.northshoresignco.com/api/quickbooks/callback as an authorized redirect URI in your Intuit app settings. This must be an exact match including https.

1
Update QB_REDIRECT_URI env var in Vercel to https://app.northshoresignco.com/api/quickbooks/callback
2
Add the same URL in developer.intuit.com → your app → Keys & OAuth → Redirect URIs
3
Review /api/quickbooks/connect — confirm it builds the Intuit OAuth URL with correct scopes: com.intuit.quickbooks.accounting
4
Review /api/quickbooks/callback — confirm it exchanges code for tokens, stores encrypted in QBOConnection table, sets QB_REALM_ID env var or saves realmId to DB
5
Add token refresh helper — before every QB API call check if accessToken expires within 5 minutes, refresh automatically using refreshToken, update QBOConnection record
6
Test: Admin visits /admin/settings/quickbooks → clicks Connect → completes Intuit OAuth → page shows "Connected — [Company Name]"

Step 2 — Auto-invoice on order completion

1
When field tech marks job complete OR admin sets order status to COMPLETED — check if QB is connected (QBOConnection exists)
2
Check: does an Invoice already exist for this order? If yes, skip. Prevents duplicates.
3
Look up QBO Customer by realtor email. If not found, create a new QBO Customer and store qboCustomerId on the User record.
4
Create QBO Invoice: line item = order type + address, amount from service pricing (configurable in AppSettings), due date = 30 days from today
5
Save local Invoice record: qboInvoiceId, status = SENT, amountCents, dueDate, realtorId, orderId
6
QBO emails invoice to realtor natively — no extra email code needed for this

Step 3 — Payment status sync

  • Register webhook in QBO developer portal for Invoice events (create, update, void)
  • Build POST /api/quickbooks/webhook — verify HMAC signature using QB_WEBHOOK_VERIFIER_TOKEN, parse event, update local Invoice status
  • QBO "Paid" → local PAID, set paidAt timestamp
  • QBO "Voided" → local VOIDED
  • Add "Sync Now" button to /admin/settings/quickbooks — polls QBO for status of all open invoices
  • Add daily sync job to scheduler at 6am — auto-sync all open invoices

Service pricing configuration

Add service pricing to AppSettings so invoice amounts can be configured without code changes. Keys: price_install, price_removal, price_change — stored as cents (e.g. 4500 = $45.00). Admin configures these in /admin/settings under a new "Pricing" section.
